Video generation from single image

Generative models, Machine learning

Stylized image
Animated video

  • Date: March-May, 2019
  • Course: Convolutional Neural Networks for visual recognition
  • Project submission: click here

What is it?

Starting from a single image of a person, we created a 3 second animated video in which the main subject moves. This is similar to 'live photo' feature in iPhones. We trained our models on 100 'Cartwheel' videos from HMDB database. The model consists of a generative and style block. GANs and convolutional LSTMs were used for generative block and fast neural style transfer for stylization.

Details:

Generative block:
  • GANs have 2 neural networks - generator and discriminator. The generator takes input and generates sample output; the discriminator tries to differentiate the generated outputs from real data. Together they train each other.
  • Generator architecture has 3D downsampling convolutions with bachnorm followed by transpose 3d convolutions.
  • Discriminator consists of 3D convolutional layers with batchnorm and leakyReLU between them.
  • The output of the generator is fed back into it 31 times to get the 32 frames of the video.
  • Similar to the generator with an extra LSTM layer after the last downsampling.
  • The final layer is a sigmoid is multiplied by 255 to get the image.
  • The model is used as a next frame predictor. During training time takes in 32 images and outputs 32 images of the same size. During test time a single image is inputted and the output is fed back in to to get the next image.
Style block
  • Takes two images — content and style image — and blends them together so the output image looks like the content image, but “painted” in the style of the style image.
  • Takes around half an hour to generate a stylized image for a content image.
  • Uses perceptual loss functions for training transfer network.
  • I used vgg-16 model to train transfer network.
  • Instance normalization was replaced with conditional instance normalization to train the transfer network with multiple styles at the same time, instead of training different networks for each style.
  • Training takes a couple of hours and stylized image generation takes seconds.


Autonomous food delivery using turtlebot

Robotic software development

  • Date: Jan-March 2019
  • Course: Introduction to robotic autonomy
  • Presentation: click here

What is it?

Explore city
  1. Navigate miniature city using turtlebot without colliding into obstacles.
  2. Map city using EKF SLAM.
  3. Record locations of different food items detected using mobilenet.
Deliver food
  1. Receive and process requests for delivery.
  2. Autonomously drive through the city to pick up desired items and deliver them to the goal.

Components:

Hardware and software used
  1. Lidar
  2. RGB camera
  3. Jetson microcontroller
  4. ROS - Robot Operating System
  5. Rviz - for visualization
  6. Gazebo - for simulation
Software stack
(implemented by each memeber of the team of 5)
  1. EKF SLAM for initial mapping
  2. Mobilenet trained on COCO dataset to detect and classify food items from RGB images
  3. A-star algorithm for path planning
  4. Shortest path traversal using travelling salesman approach
  5. Differential flatness controller for long range and PI controller for short range motion

Mini texting application

Software developement, Socket Programming

  • Date: Sept-Dec, 2019
  • Course: Programming Abstractions (Accelerated C++)
  • Presentation: click here

What is it?

  • I had no prior experience in networking, and wanted to explore a bit.
  • I made a TCP client-server application for windows OS using standard C++ libraries to efficiently handle text messaging.
  • One server could handle up to 100 clients simulatneously.
  • It could process asynchronous inputs from clients including accepting new connection, messages, change in individual client staus.
    • Client server included functionalities like creating and logging into accounts, switching chat partner, adding new friends, etc.

Details:

Server
  1. Socket programming using Winsock library to establish new client connections.
  2. User data bound to independednt objects for unique identification.
  3. Input texts stored if user offline.
  4. All communications made non-blocking.
Client
  1. User inputs and server communication made non-blocking using futures.
  2. Command table approach to handle special commands like help.
  3. Error handling and file compression of previous chats using lzw compression.
